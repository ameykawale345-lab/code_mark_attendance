The Problem

Current systems are
easy to manipulate

In many colleges, attendance systems are insecure. Students share codes, submit late, or mark attendance from outside the classroom.

🔗

Code Sharing

Students freely share attendance codes with absent classmates, making the entire system unreliable.

📍

No Location Check

No restriction on where attendance is submitted — students can mark from home or outside campus entirely.

⏰

No Time Window

Codes remain valid indefinitely, letting students submit attendance hours after the class ends.

♾️

Duplicate Entries

Existing systems allow multiple submissions for the same student on the same day without restriction.

Our Approach

3 simple restrictions,
1 robust solution

Instead of complex AI or biometrics, we chose a practical, deployable approach combining three core restrictions.

01

Time-Based Validation

Each attendance code expires after exactly 2 minutes. Late submissions are automatically rejected by the system.

02

One-Time Submission

The database tracks each student per date. Once attendance is recorded, no second submission is accepted for that day.

03

Network / IP Restriction

Student IP is captured via Flask and validated against the classroom's local WiFi network, ensuring physical presence.

Why this approach?

AI-based solutions like face recognition add complexity, cost, and privacy concerns. Our team identified a practical gap — most proxy attendance relies on sharing codes and submitting remotely. By closing those three loopholes specifically, we achieve strong security without over-engineering.

"Simple restrictions, correctly implemented, solve the real problem."

Technical Details

Under the hood

Built with Flask for backend logic and a structured database for secure, duplicate-proof record keeping.

attendance_records
ColumnTypeConstraint
student_id VARCHAR PRIMARY KEY
date DATE NOT NULL
status ENUM present/absent
submitted_ip VARCHAR VALIDATED
submitted_at TIMESTAMP AUTO
⏱

2-Minute Code Window

Generation timestamp is stored alongside the code. Submissions arriving after 120 seconds are instantly rejected by Flask middleware.

🌐

IP Capture via Flask

Using request.remote_addr in Flask to capture and validate the student's IP against the allowed classroom subnet.

🚫

Duplicate Prevention

Composite unique constraint on (student_id, date) ensures one record per student per day at the database level.
